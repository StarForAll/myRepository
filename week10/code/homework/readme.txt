作业描述:
        在执行fork/vfork操作后，父进程会复制所有的资源，包括文件描述符给子进程。编程验证父子进程对于相同文件操作时，是否会相互影响，即两者是会共享文件指针，还是拥有自己各自独立的指针体系。请编写两个版本（fork版和vfork版），实验结果贴图于答案区并附文字说明。源代码附件形式提交

1-9(写入文件:一次还是两次)

使用动态链接的方式进行功能的实现(function.c):
gcc -fpic -shared -o function.so function.c
gcc -o forkWrite forkWrite.c ./function.so -I./

进程调用fork后,子进程和父进程的文件描述符所对应的文件表项是共享的,这意味着子进程对文件的读写直接影响父进程的文件位移量(反之同理).[在使用exit的情况下,而不是使用_exit]
