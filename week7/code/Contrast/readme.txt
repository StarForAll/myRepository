本目录下编写的是对系统I/O和标准I/O的对比.
在系统I/O中,是直接进行I/O操作的,是无缓冲的.
在标准I/O中,分为三种:
  默认方式是全缓冲,在缓冲满缓冲区之后进行I/O操作(或者是程序结束之后);
  行缓冲:有换行标志进行I/O操作;
  无缓冲是和系统I/O操作类似的操作.

系统IO库函数的使用是说明缓冲方式是无缓冲方式.
fopen,fwrite...函数的使用说明缓冲方式是标准IO中的方式,一般情况是全缓冲,在程序结束或者是缓冲区满之后才会进行的操作.

fork():会创建一个子进程,会将所有父进程的信息复制到子进程中,所以在父进程的缓冲区还有内容也会复制下来,而使用write(STDOUT_FILENO, buf, sizeof(buf)是系统IO无缓冲方式,自然不会有缓冲内容留下.只有全缓冲方式下的内容在没有满之前会保留在缓冲区中,其他的缓存方式一般不会保留在缓冲区中.
